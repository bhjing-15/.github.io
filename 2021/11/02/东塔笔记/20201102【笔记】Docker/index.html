Docker

安装docker

环境：kali-2020.4

> sudo apt-get install docker docker-compose

常用命令

查看镜像

> docker images

搜索镜像

> docker search [关键字]

会列出仓库中的相关镜像

查看当前运行容器

> docker ps
--------------参数--------------
-a 列出所有（包括之前运行过的）容器
-q 只列容器的ID

容器为镜像只读层+读写层

运行态容器为由一个可读写的文件系统「静态容器」+ 隔离的进程空间和其中的进程构成

查看容器端口

> docker port CONTAINER-ID

使用流程

下拉镜像

> docker pull mysql

镜像由一层层只读层堆在一起

镜像通过指针层叠

运行容器

> docker run --name mysql-5.7 -p 3334:3306 -e MYSQL\_ROOT\_PASSWORD=123456 -d mysql:5.7
（注：下划线需反斜杠转义）

-------------语法----------------
docker run [参数] IMAGE [要在容器中运行的命令]
-------------参数----------------
--name="JBH-nginx"        为容器制定一个名称
-p [宿主端口]:[容器端口]        指定端口映射
-P        随机端口映射
-d        后台运行容器，返回容器ID（注：某些镜像无法后台启动如centos）
-i        交互式运行容器，通常与-t配合使用
-t        为容器分配一个伪输入终端
-e [变量名]="值"        设置环境变量
-v [主机目录]:[容器目录]        挂载目录，使容器保持持久性
--restart=no        容器退出时不重启
--restart=on-failure        容器故障退出时重启
--restart=always        容器退出时总重启

进入容器

> docker exec -it [容器名称] /bin/bash
即可进入容器内目录
root@CONTAINER-ID:
  • 进入MySQL数据库交互
> mysql -h <本机IP> -P <宿主端口> -u root -p
[输入密码...]
mysql > select version();

停止容器

> docker stop mysql-5.7（容器名称/containerID）
如需停止所有容器：
> docker stop $(docker ps -q)

删除容器

> docker rm [containerID]
如需删除所有容器：
> docker rm $(docker ps -q)

​ containerID：我们知道是宿主机在创建容器时随即分配的

删除镜像

> docker rmi -f [ImageID]

​ ImageID:唯一标志一个镜像，其数值根据该镜像的元数据配置文件采用sha256算法的计算获得。（不同主机版本相同ID一致）

DDOS

常见的DDOS攻击方式

Synflood 多个随机的源主机地址向目的主机发送SYN包

Smurf 欺骗性Ping分组（echo 请求）

Land-based 源地址和目的地址都设置为目标主机的地址

Ping of Death 一个包分成的多个片段叠加，长度大于65536字节

Teardrop 第一个包长度为N，第二个包的偏移量小于N

PingSweep ICMP Echo轮询多个主机

Pingflood 短时间内向目的主机发送大量ping包
